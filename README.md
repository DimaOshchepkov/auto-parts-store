# Auto Products

Интернет-магазин автозапчастей на Laravel 12 с клиентской витриной (Inertia + React) и административной панелью (Filament v5).

Проект демонстрирует полный цикл e-commerce базового уровня: каталог, карточка товара, корзина для гостя и авторизованного пользователя, авторизация с 2FA, а также CRUD-управление сущностями через админку.

## Что реализовано

### Клиентская часть магазина

- Главная страница с выводом товаров.
- Карточка товара по `slug`.
- Корзина с добавлением, изменением количества и удалением позиций.
- Дебаунс-обновление количества товара в корзине на фронтенде.

### Серверная часть

- Модели `Product`, `Category`, `Brand`, `Cart`, `CartItem`, `Order`, `OrderItem`, `Customer`, `Role`, `User`.
- Связи Eloquent между товарами, заказами, клиентами, корзинами и ролями.
- Сервис корзины (`CartService`) с транзакциями и кэшированием количества товаров.
- Слияние гостевой корзины с пользовательской после логина (через событие `Login`).
- API-эндпоинт сводки корзины (`/api/cart/summary`).

### Админ-панель

- Filament-ресурсы для `Products`, `Orders`, `Customers`, `Categories`, `Brands`.
- Работа с изображениями товаров через Spatie Media Library и Filament plugin.
- Автогенерация `slug`, валидация уникальности и базовые бизнес-правила в формах.

### Аутентификация и безопасность

- Laravel Fortify (логин, регистрация, восстановление пароля, подтверждение email, 2FA).
- Ограничение частоты попыток логина и 2FA challenge.
- Ролевая модель (`admin`/`user`) и ограничение доступа к админ-панели.

## Навыки, использованные в проекте

Этот блок можно использовать как “Hard Skills” для резюме.

### Backend (Laravel / PHP)

- Проектирование backend-архитектуры на Laravel 12.
- Проектирование БД под e-commerce домен (товары, заказы, корзина, клиенты, роли).
- Eloquent ORM: связи, касты, события моделей, сиды, фабрики, миграции.
- Выделение доменной логики в сервисный слой (`CartService`).
- Работа с middleware (например, токен гостевой корзины в cookie).
- Event-driven подход: обработка события логина для merge корзины.
- Реализация role-based access control (RBAC) для админ-доступа.
- Интеграция Laravel Fortify и настройка 2FA в headless-режиме.
- Реализация rate limiting для критичных auth-роутов.

### Admin / CMS

- Разработка админки на Filament v5.
- Создание и настройка Resource, Form, Table, Page компонентов.
- Связывание админ-форм с Eloquent relationship полями.
- Интеграция загрузки медиафайлов и конверсий изображений.

### Frontend (Inertia + React + TypeScript)

- Построение SPA-подобного интерфейса через Inertia.js v2 без отделения backend/frontend в разные проекты.
- Разработка страниц и компонентов на React 19 + TypeScript.
- Использование Wayfinder-роутов (`@/routes`, `@/actions`) для типобезопасной навигации и отправки форм.
- Реализация UI на Tailwind CSS v4 и компонентном слое (`ui/*`).
- Управление состоянием корзины и синхронизацией с сервером через Inertia router.

### DevOps / качество / процессы

- Контейнеризация среды через Laravel Sail (Docker).
- Автоматизация окружения и скриптов разработки через Composer/NPM.
- Линтинг и форматирование: ESLint, Prettier, Laravel Pint.
- Автотесты на Pest/PHPUnit для auth и settings сценариев.

## Краткий обзор: как обращаться с проектом

### 1. Требования

- Docker + Docker Compose
- Node.js + npm
- PHP/Composer локально не обязательны, если всё запускается через Sail

### 2. Быстрый старт

```bash
cp .env.example .env
vendor/bin/sail up -d
vendor/bin/sail composer install
vendor/bin/sail npm install
vendor/bin/sail artisan key:generate
vendor/bin/sail artisan migrate --seed --no-interaction
vendor/bin/sail npm run dev
```

Если нужен production-like build:

```bash
vendor/bin/sail npm run build
```

### 3. Основные команды

- Запуск контейнеров: `vendor/bin/sail up -d`
- Остановка контейнеров: `vendor/bin/sail stop`
- Запуск тестов: `vendor/bin/sail artisan test --compact`
- Форматирование PHP: `vendor/bin/sail bin pint --dirty --format agent`
- Линт фронтенда: `vendor/bin/sail npm run lint`

### 4. Где что находится

- `app/Http/Controllers` — HTTP-контроллеры (витрина, корзина, настройки)
- `app/Services` — бизнес-логика (сервис корзины)
- `app/Filament` — админ-панель и ресурсы Filament
- `app/Models` — модели и связи домена
- `resources/js/pages` — Inertia/React страницы
- `resources/js/components` — UI и бизнес-компоненты фронтенда
- `routes/` — web/api/settings роуты
- `tests/Feature` — функциональные тесты

### 5. Базовые рабочие сценарии

#### Как добавить товар в каталог

- Зайти в Filament admin panel.
- Открыть `Products`.
- Создать запись, заполнить поля и добавить изображения.

#### Как проверить работу корзины

- Открыть главную страницу.
- Добавить товары в корзину.
- Перейти в `/cart`, изменить количество, удалить позицию.

#### Как проверить авторизацию и 2FA

- Зарегистрировать/авторизовать пользователя.
- Перейти в `settings/two-factor` и включить двухфакторную аутентификацию.
